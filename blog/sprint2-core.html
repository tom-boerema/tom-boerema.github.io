Long Term Goals

What is your long term goal and or career pathway?

My career goal is to build several companies to solve real-world problems and influence the world in a positive way.

In regard to web development, this can be broken down into several more specific goals:

  1. Build a broad knowledge and competence in web development

    2. I would like to become a competent developer and have a broad knowledge of how web and software development works, including how the various components interact with each other. I would like to be able to better manage other developers as well personally as implement my own ideas into a product.

  2. Develop a high standard of habits (e.g. code structure, approach to a problem

    3. I am very passionate about efficiency which I think is highly influenced by your habits, if you build good habits, life will be much easier.

  3. Build communication skills

    4. I am quite introverted and am generally not the best in social circumstances. Building companies requires a team so communication and a positive social dynamic is paramount in growing a business. I would also like to be able to communicate on a technical level with other developers. I am not interested in being an employee and would rather hold a managment position which has an essential communication component to it.

  4. Learn React Native

    5. Although it is beyond the scope of this course, I would like to learn React Native for mobile development.

My Commitment

Based on the above description, a commitment to how you will manage your workload in this programme.

This should include - an explanation of how you will manage yourself to work productively and safely with other learners, facilitators and industry/community representatives.

A commitment as to how and where you will seek help in a timely way.

To manage my workload, I have created a schedule using the time-boxes so I have an overview of what needs to be done and when I need to do it. You can find it here: Dev Schedule

I have a couple of businesses which require upkeep so I need to be develop a habit of discipline and time/task management so I am not overwhelmed or lose focus.

Productive work is done when an efficient workflow is established which allows time and effort to be saved as well as allowing focus to be applied to the task at hand rather than workflow problems. My workflow currently includes the following elements but I am always open to improvements.

  • Communication - Slack
  • File sharing - Google Drive
  • Version Control - GitHub
  • Design - Sketch or Photoshop
  • Time tracking - Toggle
  • Task management - ClickUp or GitHub
  • Command Line - iTerm2 with custom Oh My zsh shell
  • Laptop - Macbook Pro with Better Touch Tool, Alfred
  • Text Editor - VS Code with various plugins such as Live Server

When working with others, I think it is important to develop a system and workflow early on and ensure everyone understands their responsibilities. I'm not really sure what it means by safe work but I assume it is associated with ensuring team dynamics are positive which comes down to clear communication and discussing any issues you have openly. I am a fan of being direct.

I will seek help by first searching for an answer on the internet, followed by asking other students, posting in the appropriate Slack channel or asking teachers in class.
