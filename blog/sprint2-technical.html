What are the differences between relative, absolute, and fixed positioning in CSS?

20 June 2019

I'm fixed!
I'm relative!

The position property defines the positioning method used for an element. There are actually five different position values which are: static, relative, absolute, and sticky.

I'm sticky

Static

By default, elements are position: static which is based on it's current position.
I'm static!

Relative

An element with position: relative is placed in a position that is relative to it's current position, without changing the layout around it.

Absolute

An element with position: absolute is placed in a position that is relative to it's closest positioned ancestor. Other elements will behave as if it is not there. I.e. it is inderpendent of other elements. It also has four positioning properties which are: top, right, bottom, left.

Fixed

An element with position: fixed is placed in a position that is relative to the page.

Sticky

An element with position: sticky switches between relative and fixed, depending on the user's scroll position. It is initially position: relative until the scroll reaches a certain position and it switches to position: fixed. An example of this is freezing rows in a Spreadsheet.
